# 架构最高指导原则：万物皆树

**状态**：不可违背
**适用范围**：Distill 所有核心业务逻辑层

## 1. 唯一法则：统一中间表示

在 Distill 的世界里，不存在“文件”，不存在“文件夹”，也不存在“压缩包”。
系统内部唯一认可的实体只有一种：**虚拟树**。
无论外界输入的物理形态多么混乱，在跨过系统边界进入引擎的那一瞬间，必须被坍缩为一棵标准的、由节点和哈希值构成的虚拟树。

## 2. 形态坍缩机制

不同类型的输入，必须通过特定的“适配器”完成向虚拟树的转换，但其本质动作只有两种：

* **单文件的“向上组装”**：将孤立的单个文件，包装为一个没有父节点的、极度扁平的虚拟树。
* **压缩包/目录的“向下拆分”**：无视外层物理包裹，将内部的嵌套结构逐层剥离、映射，降维展开为一棵完整的虚拟树。
**结论**：经过组装或拆分后，单文件、文件夹、ZIP 包在引擎眼中具有完全等同的数据结构，没有任何区别。

## 3. 系统边界划定

```text
[ 现实混沌域 ]                  [ 纯粹秩序域 ]
               |                 |
 单个 PDF  --->| (FileAdapter)   |
 散装文件夹 -->| (DirAdapter)  --|--->  只有“虚拟树”
 嵌套 ZIP  --->| (ZipAdapter)   |      引擎只对树进行：
               |                 |      - 哈希比对
               -------------------      - 冗余判定
                                       - 清单生成
                                       - 逆向组装

```

**原则**：在“纯粹秩序域”内，绝对禁止出现任何针对物理文件格式的判断（严禁出现 `if isZip`）。引擎必须是一个“瞎子”。

## 4. 违背原则的代码嗅探

如果你在代码审查时看到以下模式，说明架构已经被污染，必须立刻重构：

* ❌ 在 `internal/engine/` 目录下的任何文件中引入了 `archive/zip` 包。
* ❌ 在处理清单或去重逻辑中使用了 `filepath.Ext()` 来判断后缀。
* ❌ 为了处理单文件，在主干流程里写了一个特殊的 `if depth == 0` 分支。
